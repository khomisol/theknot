# Workflow: Document Project

## Purpose
Analyze and document existing (brownfield) codebase.

## Agent
Analyst (Mary) or Tech Writer (Paige)

## Prerequisites
- Existing codebase to analyze
- Access to source files

---

## Process

### Step 1: Scan Structure

**Actions:**
1. Map directory structure
2. Identify major components
3. Note technology stack
4. Find configuration files

---

### Step 2: Analyze Architecture

**Document:**
- System components
- Data flow
- External integrations
- Database schema

---

### Step 3: Identify Patterns

**Look for:**
- Coding conventions
- Naming patterns
- File organization
- Testing approach

---

### Step 4: Document Findings

**Create:** `docs/index.md`

**Include:**
- Project overview
- Architecture summary
- Component descriptions
- Conventions guide
- Setup instructions

---

## Output

- `docs/index.md` - Project documentation
- Understanding of existing codebase
- Foundation for new development

---

## Next Steps

After documenting:
- Create PRD for new features (PM)
- Design architecture additions (Architect)
- Begin implementation (Developer)
