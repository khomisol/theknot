# Workflow: Retrospective

## Purpose
Review completed epic, extract lessons, and improve process.

## Agent
Scrum Master (Bob)

## Prerequisites
- Epic complete (all stories done)
- Team available for reflection

---

## Process

### Step 1: Gather Data

**Review:**
1. All completed stories in the epic
2. Dev Notes from each story
3. Code review findings
4. Any issues encountered

---

### Step 2: What Went Well

**Discuss:**
- Successes and wins
- Effective practices
- Good decisions
- Team strengths

**Capture:** List of positives to continue

---

### Step 3: What Could Improve

**Discuss:**
- Challenges faced
- Inefficiencies
- Surprises
- Pain points

**Capture:** List of areas for improvement

---

### Step 4: Action Items

**For each improvement area:**
- Specific action to take
- Who owns it
- When to implement

**Format:**
```markdown
## Action Items

1. **[Action]**
   - Owner: [Name]
   - Due: [Date]
   - Status: [ ] Pending

2. **[Action]**
   - Owner: [Name]
   - Due: [Date]
   - Status: [ ] Pending
```

---

### Step 5: Document

**Create retrospective notes:**

```markdown
# Retrospective: Epic [N] - [Name]

**Date:** [Date]
**Participants:** [Names]

## Summary
[Brief summary of the epic]

## What Went Well
- [Item 1]
- [Item 2]
- [Item 3]

## What Could Improve
- [Item 1]
- [Item 2]
- [Item 3]

## Action Items
[As above]

## Lessons Learned
[Key takeaways for future epics]
```

---

## Output

- Retrospective notes
- Action items with owners
- Lessons for next epic

---

## Tips

1. **Be honest** - Safe space for feedback
2. **Be specific** - Vague feedback doesn't help
3. **Be constructive** - Focus on improvement
4. **Follow through** - Action items matter

---

## Next Steps

After retrospective:
- Implement action items
- Apply lessons to next epic
- Begin next epic's stories
