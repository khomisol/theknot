# Story Draft Validation Checklist

Use this checklist to validate story quality before development.

---

## Story Format

- [ ] User story follows "As a... I want... So that..." format
- [ ] Story is from user's perspective (not technical)
- [ ] Benefit/value is clear
- [ ] Story is appropriately sized (completable in sprint)

## Acceptance Criteria

- [ ] All acceptance criteria are testable
- [ ] Criteria are specific (not vague)
- [ ] Criteria cover happy path
- [ ] Criteria cover error scenarios
- [ ] No missing edge cases
- [ ] Criteria are independent of implementation

## Tasks

- [ ] Tasks are properly sequenced
- [ ] Each task is actionable
- [ ] Subtasks provide sufficient detail
- [ ] Test tasks are included
- [ ] No missing steps

## Technical Context

- [ ] Architecture reference included
- [ ] Dependencies identified
- [ ] Affected files listed
- [ ] API changes documented (if any)
- [ ] Database changes documented (if any)

## Test Requirements

- [ ] Unit test requirements specified
- [ ] Integration test requirements specified
- [ ] Edge cases identified
- [ ] Test coverage expectations clear

## Dependencies

- [ ] Blocking dependencies identified
- [ ] Dependency status known
- [ ] No circular dependencies

## Clarity

- [ ] No ambiguous language
- [ ] Technical terms defined or clear
- [ ] Story is self-contained
- [ ] Developer can start without questions
